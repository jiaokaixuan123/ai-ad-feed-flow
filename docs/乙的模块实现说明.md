## 3. 曝光埋点（Day 11 核心任务）

**口径**：可见面积 ≥50% 且持续 ≥500ms 算一次曝光（去重逻辑由甲在视图层实现）。

### 乙提供的数据层调用链

```
甲（视图层判断可见）
  └─ viewModel.recordImpression(adId)
       └─ repository.recordImpression(id)
            └─ interactionStore.recordImpression(adId)
                 └─ state.impressionCount + 1
```

### 甲需要做的事（视图层）

在 `FeedFragment`（或 `RecyclerView` 滚动监听）里，当卡片首次满足曝光口径时调用：

```kotlin
viewModel.recordImpression(card.ad.id)
```

去重由甲维护一个已曝光 id 的 `Set<String>`，避免重复上报（或直接依赖 `impressionCount > 0` 判断）。

## 4. 数据层异步化

### 4.1 FeedDataSource 接口（suspend）

```kotlin
interface FeedDataSource {
    suspend fun getPage(channel: FeedChannel, page: Int, pageSize: Int): PageResult<AdItem>
    suspend fun getById(id: String): AdItem?
}
```

`MockFeedDataSource` 直接返回（无实际 I/O），`RetrofitFeedDataSource` 调用 Retrofit suspend 接口。

### 4.2 FeedRepository（withContext IO）

```kotlin
suspend fun loadPage(...): PageResult<FeedCardUiModel> = withContext(Dispatchers.IO) { ... }
suspend fun getCard(id: String): FeedCardUiModel? = withContext(Dispatchers.IO) { ... }
```

### 4.3 FeedViewModel / DetailViewModel（viewModelScope.launch）

```kotlin
fun loadFirstPage() {
    viewModelScope.launch {
        _uiState.update { it.copy(isRefreshing = true) }
        try {
            val result = repository.loadPage(...)
            _uiState.update { it.copy(items = result.items, isRefreshing = false) }
        } catch (e: Exception) {
            _uiState.update { it.copy(isRefreshing = false, errorMessage = e.message) }
        }
    }
}
```

错误时 `FeedUiState.errorMessage` 有值，甲在 `render()` 里判断并显示错误态 UI 即可。

## 6. ExoPlayer 播放器池 — PlayerPool

文件路径：[player/PlayerPool.kt](../app/src/main/java/com/example/ai_ad_feed_flow/player/PlayerPool.kt)

### 设计要点

- 维护一个最多 2 个 ExoPlayer 实例的池，避免每个视频卡片创建独立实例
- 内置 100MB `SimpleCache` 磁盘缓存（OkHttp 数据源）
- 默认静音（`volume = 0f`），点击后由甲的 UI 逻辑解除静音

### 甲在 VideoAdViewHolder 里的调用方式

```kotlin
// onViewAttachedToWindow 或 bind() 时绑定
val player = AppGraph.playerPool.bindPlayer(this, card.ad.videoUrl!!)
binding.playerView.player = player
player.play()

// onViewDetachedFromWindow 或视频滑出屏幕时解绑
AppGraph.playerPool.unbindPlayer(this)
binding.playerView.player = null
```

`holderKey` 传 ViewHolder 实例（`this`）即可，池内部以此作为 key 做 attach/detach 映射。

### 生命周期（乙负责）

| 时机 | 操作 |
|------|------|
| App 启动 | `AppGraph.init(context)` 创建 PlayerPool（已在 `App.kt` 完成） |
| 视频卡片 attach | `AppGraph.playerPool.bindPlayer(holder, url)` |
| 视频卡片 detach | `AppGraph.playerPool.unbindPlayer(holder)` |
| Activity/Fragment destroy | `AppGraph.playerPool.release()`（如需，甲处理时机） |

## 8. 后端服务

目录：[backend/](../backend/)

### 8.1 启动

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

启动后 API 文档：http://localhost:8000/docs

### 8.2 接口

#### `GET /feed`

```
参数：channel=featured|ecom|local  page=1  size=20
响应：{ "page": 1, "hasMore": true, "items": [...] }
```

每条 item 已内嵌 AI 预生成的 `summary` 和 `tags`（缓存缺失时降级回标题/空列表）。

#### `POST /search`

```json
请求：{ "query": "适合学生的平价运动鞋" }
响应：{ "results": [{ "id": "ad_ecom_01", "reason": "平价+运动+学生" }] }
```

需要在环境变量中配置 `DASHSCOPE_API_KEY`，否则返回 503。

### 8.3 AI 摘要/标签预生成

```bash
export DASHSCOPE_API_KEY=sk-xxx   # 阿里云百炼控制台获取
python generate_ai.py
```

脚本特性：

- **幂等**：已缓存的 ad_id 直接跳过，不重复消耗 token
- **schema 校验**：输出必须包含 `summary`（字符串）和 `tags`（非空列表），否则重试
- **最多重试 3 次**，全部失败则跳过（接口降级）
- 每条成功后立即写入 `ai_cache.json`，防止中断丢失进度
- 最终将缓存注入 `data.py` 的 `AI_CACHE` 字典，随 `/feed` 接口下发
